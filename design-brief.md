# 아이젠하워 투두 — 디자인 브리프

## 프로젝트 개요

- **앱 이름:** 아이젠하워 투두
- **한 줄 설명:** 할 일을 긴급/중요 4사분면으로 분류하고, 내 시간 사용 패턴을 보여주는 투두 앱
- **타겟:** 20~30대 한국 직장인/대학생
- **플랫폼:** 모바일 퍼스트 반응형 웹앱 (375px 기준, 데스크톱 대응)
- **기술 스택:** Next.js + Tailwind CSS + dnd-kit (드래그앤드롭)
- **핵심 명언:** "긴급한 것은 중요하지 않고, 중요한 것은 긴급하지 않다." — 아이젠하워

## 디자인 방향

### 톤 & 무드

- **다크 테마 기반** (배경 #0f0f13 ~ #18181b 계열)
- 각 사분면이 고유 색상 글로우로 구분되는 시각적 정체성
- 깔끔하고 현대적, 미니멀하면서 **살아있는 느낌** — 구체적 스펙:
  - 모든 전환: Framer Motion spring (stiffness: 300, damping: 20, mass: 1)
  - 카드 호버/터치: 미세한 글로우 (사분면 색상 box-shadow 0 0 8px, opacity 0.3)
  - 탭 전환: 좌우 슬라이드 200ms ease-out
  - 숫자 변경 (카운트, 달성률): 바운스 스케일 애니메이션
  - "차갑지 않은" = 정적이지 않은. 배경색은 중립 유지, 인터랙션에서 온기 전달
- **인터랙션이 핵심** — 드래그 시 카드가 살짝 기울어지고, 확대되고, 그림자가 깊어지는 물리적 조작감
- 앱을 열었을 때 "와 이거 기분 좋다" 느낌이 첫인상

### 색상 시스템

| 사분면 | 이름 | 라벨 | 주색상 | 용도 |
|--------|------|------|--------|------|
| Q1 (좌상) | 즉시 실행 (DO) | 긴급 + 중요 | 🔴 빨강 (#f87171 ~ #ef4444) | 테두리, 라벨, 글로우 |
| Q2 (우상) | 계획 수립 (PLAN) | 중요 + 안긴급 | 🔵 파랑 (#60a5fa ~ #3b82f6) | 테두리, 라벨, 글로우 |
| Q3 (좌하) | 위임/최소화 (DELEGATE) | 긴급 + 안중요 | 🟡 노랑 (#fbbf24 ~ #f59e0b) | 테두리, 라벨, 글로우 |
| Q4 (우하) | 제거 (DELETE) | 안긴급 + 안중요 | ⚫ 회색 (#71717a ~ #52525b) | 테두리, 라벨, 글로우 |

- 사분면 배경: 각 색상의 극도로 어두운 버전 (투명도 5~10%)
- 카드 배경: rgba(255, 255, 255, 0.05) — 반투명 유리 느낌
- 텍스트: #e4e4e7 (본문), #d4d4d8 (카드), #52525b (메타/서브)

**디자인 요구사항:**
- 2x2 CSS Grid, gap 8px, padding 12px
- 각 사분면: border-radius 16px, 1px solid border (사분면 색상)
- **시각적 계층 (Q2 강조):** Q2(PLAN)의 배경 tint를 10-15%로 다른 사분면(5-10%)보다 강하게, 테두리 두께 1.5px, 글로우 효과 추가. Q4(DELETE)는 가장 어둡게(3-5% tint). Q1은 색상 강도로 긴급함을 표현하되 Q2가 시각적으로 "여기가 중요합니다"를 전달. 아이젠하워 철학: "중요한 것에 집중하라"
- 사분면 헤더: **한국어 메인** — '즉시 실행 · 2개' (메인, 13px semibold) / 'DO' (서브, 9px uppercase, opacity 0.5). 영어 라벨은 사분면 색상과 동일한 작은 텍스트로 우상단 배치
- 할 일 카드: border-radius 10px, 13px 폰트, 내부 패딩 10px 12px
- **드래그 중 카드:** scale(1.04), rotate(-1.5deg), 깊은 그림자, 테두리 밝아짐
- **드롭 대상 사분면:** 테두리 색상 강해짐, 배경 살짝 밝아짐
- FAB 버튼: 52x52, border-radius 16px, 파란 그라디언트, 그림자
- 하단 탭바: fixed, 아이콘 + 라벨, 활성 탭 밝은 색

## 인터랙션 상세

### 드래그 앤 드롭 (가장 중요한 인터랙션)

1. **터치/클릭 시작:** 카드에 살짝 그림자 추가 (0.1초)
2. **드래그 중:**
   - 카드: scale(1.04), rotate(-1.5deg), 깊은 box-shadow
   - 원래 위치: 빈 공간 표시 (점선 또는 투명 플레이스홀더)
   - 대상 사분면: 테두리 색상 강해짐, 배경 미세하게 밝아짐
   - 다른 카드: 자연스럽게 빈 공간 만들며 재배치 (스프링 애니메이션)
3. **드롭 시:**
   - 카드: 스프링 바운스로 착지 (Framer Motion spring)
   - 짧은 햅틱 피드백 (모바일)
   - 사분면 색상 잠깐 펄스
4. **취소 (원래 위치로):** 부드럽게 원위치 복귀

- **매트릭스 뷰:** 완료 후 2초 동안 체크마크 + 취소선 보여주고, 서서히 사라짐 (opacity 1→0, 300ms). 사분면 헤더의 완료 카운트에 +1 바운스 애니메이션. 데이터는 localStorage에 유지 (마크다운 내보내기 시 `- [x]`로 포함)
- **캘린더 뷰:** 완료된 할 일은 취소선 + opacity 0.4로 유지 (날짜별 기록 확인 목적)
- **주간 리포트:** 완료된 할 일도 사분면 비율 계산에 포함

### 탭 전환

- 캘린더 ↔ 매트릭스 ↔ 리포트: 좌우 슬라이드 트랜지션

## 미해결 디자인 결정 — 해결

### 스프링 애니메이션 물리값
- 드래그 카드 착지: stiffness 300, damping 20, mass 1 (자연스러운 바운스)
- 카드 등장: stiffness 400, damping 25 (빠르고 경쾌)
- 탭 전환: duration 200ms, ease-out

### 캘린더 도트 오버플로우
- 하루에 할 일이 여러 사분면에 걸칠 경우: 최대 4개 도트 가로 배치 (각 3px, gap 1px)
- 4개 초과 시에도 4개 도트만 표시 (각 사분면당 최대 1개)

### FAB 그라디언트
- 방향: 좌상→우하 (135deg)
- 색상: #3b82f6 → #2563eb
- 그림자: 0 4px 12px rgba(59, 130, 246, 0.4)

### 바텀시트 safe area
- padding-bottom: env(safe-area-inset-bottom) 적용
- z-index: 탭바(50) < 바텀시트(100) < 오버레이(90)
- 바텀시트 열릴 때: 반투명 오버레이 (rgba(0,0,0,0.5)) + 탭 시 닫기

### 리포트 주간 내비게이션
- 리포트 상단에 ◀ ▶ 화살표로 이전/다음 주 이동
- 현재 주가 기본, 최대 4주 전까지 조회 가능

## 핵심 원칙

1. **드래그 느낌이 전부다** — 드래그 인터랙션이 "기분 좋다"고 느껴져야 함
2. **사분면 색상이 정체성** — 빨강/파랑/노랑/회색이 앱 전체를 관통
3. **한국어 네이티브** — 영어 라벨(DO/PLAN/DELEGATE/DELETE)은 서브, 한국어가 메인
4. **공유 카드는 SVG 필터 없이** — html2canvas 호환성을 위해 순수 CSS만 사용
5. **다크 테마 일관성** — 모든 화면에서 동일한 다크 톤 유지
6. **이모지 금지** — 사분면 표시는 CSS 컬러 도트(8px 원), 탭바는 SVG 아이콘, 반복 표시는 텍스트 뱃지로. 이모지는 AI틱한 느낌을 주므로 사용하지 않음
