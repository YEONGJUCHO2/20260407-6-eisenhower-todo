# Design Spec: 반복 세부 옵션 + 시간 피커 + 디자인 시스템 업그레이드

## Overview

아이젠하워 투두 앱에 3가지 변경을 적용한다:
1. **반복 할 일 세부 옵션 UI** — 매주 요일 선택, 매월 날짜 선택
2. **시간 범위 피커** — 시작~종료 시간 입력
3. **디자인 시스템 업그레이드** — Notion 기반 라이트/다크 모드, 매트릭스 축 화살표, 앱 로고, 온보딩 개선

---

## 1. 시간 범위 피커

### 데이터 모델 변경

`Todo` 인터페이스에 필드 추가:

```typescript
startTime?: string   // "HH:mm" (예: "09:00")
endTime?: string     // "HH:mm" (예: "11:00")
```

- 선택사항 — 시간을 설정하지 않으면 undefined
- 30분 단위 (00, 30)

### UI 스타일: 인라인 숫자 입력 (C안)

시/분 박스를 탭하면 숫자가 조절되는 컴팩트 방식.

```
[09]:[00]  →  [11]:[00]
 블루          레드
```

- 시작 시간: quadrant-plan-primary (#adc6ff / 다크, #0054b3 / 라이트)
- 종료 시간: quadrant-do-primary (#ffb3ad / 다크, #c0342f / 라이트)
- 박스 배경: surface-container-high
- 구분자: "→" in outline 색상
- 탭하면 ±30분 조절, 길게 누르면 연속 변경

## 2. 반복 세부 옵션

### 매주 요일 선택: 원형 칩 일렬 (A안)

7개 원형 버튼 한 줄, 탭하면 토글 (다중 선택).

```
(월) (화) (수) (목) (금) (토) (일)
 ●        ●        ●
```

- 선택됨: quadrant-plan-container (#0566d9) bg, white text, font-weight 600
- 미선택: surface-container-high bg, outline text
- 크기: 30x30px (모달 내), 38x38px (넉넉한 공간)
- 데이터: `repeatDays: number[]` (0=일, 1=월 ... 6=토)

### 매월 날짜 선택: 인라인 숫자 (B안)

시간 피커와 동일한 ▲▼ 스타일로 일관성 확보.

```
매월  [15]일
       ▲▼
```

- 숫자 색상: quadrant-plan-primary
- 범위: 1~31
- 데이터: `repeatDate: number`

## 3. AddTodoModal 레이아웃: 접이식 섹션 (A안)

기존 필드(제목, 사분면, 날짜) 아래에 2개 접이식 섹션 추가.

### 구조

```
┌─────────────────────────┐
│ 할 일을 입력하세요        │  ← 제목 인풋
├─────────────────────────┤
│ [DO] [PLAN]             │  ← 사분면 2x2
│ [DELEGATE] [DELETE]     │
├─────────────────────────┤
│ 📅 2026년 4월 8일 (수)   │  ← 날짜
├─────────────────────────┤
│ ⏰ 시간 설정          ▼  │  ← 접이식 (기본 접힘)
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄ │
│  [09]:[00] → [11]:[00]  │  ← 펼침 시 노출
├─────────────────────────┤
│ 🔁 반복: 없음         ▼  │  ← 접이식 (기본 접힘)
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄ │
│ [없음][매일][매주][매월]  │  ← 펼침 시: 타입 칩
│ (월)(화)(수)(목)(금)...   │  ← 매주 선택 시: 요일 칩
├─────────────────────────┤
│      [ 추가하기 ]        │
└─────────────────────────┘
```

### 접이식 동작
- 기본 상태: 접혀있음 (한 줄 요약만 표시)
- 탭하면 펼침: ease-out 200ms 애니메이션
- 시간 설정됨 → 접힌 상태에서 "⏰ 09:00 → 11:00" 표시
- 반복 설정됨 → 접힌 상태에서 "🔁 매주 · 월,수,금" 표시

## 6. 디자인 시스템 업그레이드

### 라이트/다크 모드

Notion의 따뜻한 뉴트럴 기반으로 라이트 모드 추가. 다크 모드는 기존 유지.

**라이트 모드 Surface Scale:**

| Token | Value | Role |
|-------|-------|------|
| surface | #ffffff | 앱 배경 |
| surface-container-low | #f6f5f4 | 카드/사분면 배경 (warm white) |
| surface-container | #eeedeb | 모달/시트 배경 |
| surface-container-high | #e8e7e5 | 입력 필드 배경 |
| surface-container-highest | #dddbd8 | 활성 요소 배경 |

**라이트 모드 Text:**

| Token | Value | Role |
|-------|-------|------|
| on-surface | rgba(0,0,0,0.95) | 본문 (순수 검정이 아닌 근검정) |
| on-surface-variant | #615d59 | 보조 텍스트 |
| outline | #a39e98 | 비활성 텍스트 |

**라이트 모드 Border:**
- whisper border: `1px solid rgba(0,0,0,0.1)` (Notion 스타일)

**라이트 모드 사분면 색상 (진한 톤):**

| Quadrant | Primary | Container |
|----------|---------|-----------|
| DO | #c0342f | #e03e3b |
| PLAN | #0054b3 | #0566d9 |
| DELEGATE | #9a6400 | #b37300 |
| DELETE | #555a6b | #6b7085 |

**다크 모드 사분면 색상 (기존 유지):**

| Quadrant | Primary | Container |
|----------|---------|-----------|
| DO | #ffb3ad | #ff5451 |
| PLAN | #adc6ff | #0566d9 |
| DELEGATE | #ffb95f | #ca8100 |
| DELETE | #8c909f | #424754 |

### 테마 전환
- 시스템 설정 따름 (`prefers-color-scheme`) 기본값
- 설정 화면에서 수동 전환 가능 (시스템/라이트/다크)
- CSS 변수 기반 토큰으로 전환

## 7. 매트릭스 축 그라디언트 화살표

매트릭스 뷰에 긴급/중요 축 표시.

### 방향
- **가로 ←**: 왼쪽 = 긴급. 오른쪽(투명/가늘) → 왼쪽(진하고/굵음)
- **세로 ↑**: 위쪽 = 중요. 아래(투명/가늘) → 위(진하고/굵음)

### 라벨 배치
- "긴급": 가로 화살표 **위쪽**, 중앙 배치
- "중요": 세로 화살표 **왼쪽**, 90도 눕혀서 중앙 배치

### 스타일
- SVG polygon으로 두께 변화 (가늘→굵음)
- 그라디언트: opacity 0 → 0.12 → 0.3 → 0.55
- 화살촉: 삼각형, opacity 0.55~0.65
- 다크: DO red (#ffb3ad) 가로, PLAN blue (#adc6ff) 세로
- 라이트: DO red (#c0342f) 가로, PLAN blue (#0054b3) 세로

### 적용 위치
- 매트릭스 뷰 (항상 표시)
- 온보딩 1단계

## 8. 앱 로고: 미니 컬러 그리드

4색 사각형 2x2 그리드.

```
┌──┐┌──┐
│🔴││🔵│   ← 4사분면 색상
└──┘└──┘
┌──┐┌──┐
│🟡││⚫│
└──┘└──┘
```

### 스펙
- 크기: 각 사각형 8x8px, gap 1.5px, 전체 약 18x18px (헤더용)
- border-radius: 3px 각 사각형
- 색상: 각 사분면 primary의 25% opacity (다크) / 20% opacity (라이트)

### 적용 위치
- 헤더 왼쪽 상단 (앱 로고)
- 온보딩 타이틀 위 (큰 사이즈: 각 24x24px, gap 3px)

---

## 9. 온보딩 1단계 개선

현재 텍스트+아이콘만 있는 1단계를 시각적으로 개선.

### 구성 (위→아래)
1. **미니 컬러 그리드** (큰 사이즈) — 이모지 대체
2. **"아이젠하워 매트릭스"** 타이틀
3. **서브 텍스트** — "할 일을 긴급/중요로 나눠 4가지로 분류하세요"
4. **매트릭스 다이어그램** — 그라디언트 화살표 + 2x2 사분면 (각 칸에 짧은 행동 지침)
5. **PLAN 강조 메시지:**
   ```
   💡 가장 중요한 건 PLAN
   — 여기가 인생을 바꾸는 곳입니다
   ```
6. Progress dots + 다음/건너뛰기 버튼
